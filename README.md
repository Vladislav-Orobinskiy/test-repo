# test-repo

## Описание

Проект для разбора почтового лога, загрузки данных в PostgreSQL и организации поиска по адресу получателя.

## Начало работы

Клонируйте репозиторий:
```bash
git clone https://github.com/Vladislav-Orobinskiy/test-repo.git
cd it-1
```

## Установка и запуск

### 1. Запуск Docker контейнеров

Запустите PostgreSQL и веб-сервер:
```bash
docker-compose up -d
```

Это запустит:
- PostgreSQL на порту 5432
- Веб-сервер на порту 8080

### 2. Загрузка данных из maillog

Поместите файл `maillog` в корневую директорию проекта и выполните:

```bash
docker exec -it test-repo-web perl /app/parse_maillog.pl /app/maillog
```

Или если файл находится локально:
```bash
perl parse_maillog.pl maillog
```

Скрипт разберет файл лога и загрузит данные в базу:
- Строки с флагом `<=` (прибытие сообщения) → таблица `message`
- Остальные строки → таблица `log`

### 3. Использование веб-интерфейса

Откройте в браузере:
```
http://localhost:8080/index.html
```

Введите адрес получателя в форму поиска. Результаты будут отсортированы по идентификаторам сообщений (int_id) и времени их появления.

## Структура проекта

- `docker-compose.yml` - конфигурация Docker контейнеров
- `Dockerfile` - образ для веб-сервера
- `init.sql` - схема базы данных (таблицы message и log)
- `parse_maillog.pl` - скрипт для парсинга maillog и загрузки в БД
- `search.psgi` - PSGI приложение для обработки поисковых запросов
- `app.psgi` - главное Plack приложение для веб-сервера
- `index.html` - HTML страница с формой поиска
- `tmpl/search.html` - HTML шаблон для страницы поиска

## Подключение к базе данных

**Параметры подключения:**
- Host: localhost (или `postgres` из контейнера)
- Port: 5432
- User: postgres
- Password: postgres
- Database: testdb

**Через psql:**
```bash
docker exec -it test-repo-postgres psql -U postgres -d testdb
```

## Формат maillog

Файл лога должен иметь формат:
```
дата время int_id флаг адрес остальная_информация
```

Флаги:
- `<=` - прибытие сообщения (отправитель)
- `=>` - нормальная доставка
- `->` - дополнительный адрес
- `**` - доставка не удалась
- `==` - доставка задержана
